🧠 提示范式转变

❌ 旧范式（GPT-4 / Sonnet 有效）

  • 写超长超详细的提示词
  • 微观管理每一步实现
  • 提供大量规范和限制
  • 分步骤逐步引导模型
  • 只用于大任务

✅ 新范式（Fable 5）

  • 对目标模糊，对外观清晰
  • 给目标 + 约束，让它推理
  • 删除所有 verbose scaffolding
  • Scope Guard: 最简可行方案
  • 任何有智能需求的地方都用

🚫 Guard Rail 规避技巧

  • 禁止句式 — "告诉我关于 Fable 5 的一切"会被识别为模型蒸馏攻击企图 → 自动降级
  • 不要提网络安全 — "帮我检查 App 安全性"会触发 safeguard
  • 正确做法：两段式 — 先用 Opus 4.8 做分析 → 结果包装 XML tag → 给 Fable 评估实现方案
  • XML tag 包装 — 把其他模型的输出包装到 XML tag 再给 Fable，降低触发概率
