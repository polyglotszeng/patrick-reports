01 / 系统痛点

单个 Agent 已是天花板，协作是下一条曲线

单兵 AI 在跨领域、长期记忆、并行执行时迅速触顶。真正的杠杆来自一个让多 agent 共享上下文、分工协作、彼此涌现的操作系统。

专业化

每个 agent 只在一个领域做深度研究——投资、量子、核聚变、法律、游戏设计，胜过一个什么都懂一点的通用助手。

并行化

多个 agent 同时工作。一周的研究量被压缩进一夜的 cron 任务，人只做最终决策。

涌现

10 个 agent 的圆桌讨论，产生单兵做不到的洞察。共享上下文让协作产生 1+1>2 的认知增量。

02 / 蜂群系统

共享大脑 + 蜂群协作 = 可演化的智能体网络

所有 agent 读写同一个共享上下文，配合六层记忆体系与多模型路由，构成一个能持续进化的 Agent Operating System。

01 六层记忆体系

L1 日志 → L2 周摘要 → L3 长期 → L4 休眠 → L5 角色 → L6 工具。让 agent 拥有可沉淀、可召回的长期记忆。

02 SAGE 动态记忆图

语义聚类 + 强化学习边管理 + 共享图，把零散记忆组织成可推理的知识网络。

03 多模型路由

scatter / ensemble / pipe 三种模式，按任务把请求分发到最合适的模型。

03 / 共享上下文架构

六层记忆体系：让 Agent 拥有可沉淀的长期记忆

Orchestrator 协调任务分配，所有 sub-agent 通过 ReadSharedContext / WriteSharedContext 读写共享状态。

L1
日志层 · Session Logs
每次对话的完整记录，毫秒级写入，用于回溯与审计
L2
周摘要 · Weekly Summary
自动聚合一周关键事件，压缩冗余，保留信号
L3
长期记忆 · Long-term
跨 session 持久化的事实、偏好与决策记录
L4
休眠层 · Dormant
低优先级记忆归档，需要时可被召回激活
L5
角色层 · Role Memory
每个 agent 专属的角色定义、知识库与行为约束
L6
工具层 · Tool Registry
可用工具的白名单与调用历史，确保安全与可审计
