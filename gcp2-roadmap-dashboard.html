核心 falsification (3 个)

Falsification #1
30 样本 GBDT 100% 过拟合
Exp-15 train R²=0.95, test R²<0.05, 差距 >0.9。根因: 30 季度样本远小于 5 因子 × 5 弱学习器参数。推论: 任何非线性方法在 <500 样本下都不能用。
Falsification #2
Exp-12 in-sample 0.45 → 0.95 是 overfit 上限
5 因子 vs 单因子的 R² 改善从 0.45 → 0.95 是 in-sample 复刻, 真实 OOS 应接近 0。GCP-2.0 必须强制 OOS 测试 (4-fold TS-CV)。
Falsification #3
季度数据扩无新信息
linear interpolation 与同季 rolling 强相关 (ρ >0.9), 480 样本是"工程就绪"不是"alpha 增量"。数据扩需真实高频或新数据源。

⚠️ GCP-2.0 5 Falsification 框架

Falsification #1
数据扩边际递减
Exp-12 5 因子 + 新 1 因子 (A 方向) IC 提升 < 0.01。含义: 新数据源没有 alpha 增量。应对: 收工 A 方向, 启 GCP-3.0 寻找非因子路线。
Falsification #2
Regime 切换频率低
16 季度 3 label 切换次数 < 5 次。含义: B 方向 regime 本身是 hindsight bias, 统计上无意义。应对: B 方向降级为"regime 辅助"而非主线。
Falsification #3
风险平价 vs 等权差异 < 2%
Sharpe / Max DD 改善 < 2pp。含义: C 方向简化模型陷阱, 交易成本 / regime 切换吃掉 alpha。应对: 需引入 BL (C2) 或 Kelly (C3) 才有可能突破。
Falsification #4
LLM codegen 幻觉
生成的 backtest.py 有 syntax error / 跑不通 / 输出与 baseline 偏差 >5%。含义: D 方向 LLM 不可靠。应对: 限制 LLM 用法 (D1 因子合成, D3 模板填充), 不依赖 D4 自动报告。
Falsification #5
新方向投入 ROI 不明
1 周 + Mac 资源 vs 1-3pp 改善。含义: GCP-2.0 整体方向 ROI 不明, 可能不如"接 IBKR 抓真实数据"单方向。应对: W4 末算总 ROI, < 3pp/方向 提前收工, 启 GCP-3.0。

🟡 2 个决策点 (Patrick)

决策 1 · 触发 W2 前 (8/10)
是否投 ~$10/月 IBKR 抓真实季度 OHLC?

选项 A: 投 $10/月 — IBKR paper trading + TWS API 抓 4 年日级 OHLC (~10 万样本)

选项 B: 不投, 用免费数据 — GBDT/HMM/LLM 在 <500 样本下不能用

选项 C: 找免费替代 — Tiingo / Alpha Vantage / Polygon.io

推荐: A. 投 $10/月 IBKR (决策成本低, 解锁 4 方向真实验证)
决策 2 · 触发 W4 末 (8/28)
W1-W4 失败率 > 50% 是否跳过 W5-W8 deep-dive?

判定标准: W1-W4 4 最小实验中, 失败 (falsification 命中) 多少个?

≤ 50% 失败 (≤2 个) → 启动 W5-W8 deep-dive, 每方向 3-4h

> 50% 失败 (≥3 个) → 跳过 W5-W8, 收工 GCP-2.0, 启 GCP-3.0

推荐: 看 W1-W4 实际数据决定, 不预先承诺
