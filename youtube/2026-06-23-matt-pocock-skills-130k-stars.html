🔧 Skills vs Abilities（关键分类）

维度Procedure（程序）Ability（能力）
触发者 用户（slash command） 模型（自动）
例子 `/grill-me` `/teach` `/prd` React 编码标准
大小 短（4-5 句） 长（详细）
Context leak ❌ 不 leak ✅ leak description
用户控制 高 低
Matt 偏好 ✅ 主用 ⚠️ 限制（disable invocation）
Superpowers 偏好 少 ✅ 主用
"I prefer my skills as procedures. I like to be the one in control. I know my skills, I know my abilities. I don't want to delegate my thinking to the model."

关键警告

⚠️ 100 abilities = 100 descriptions leak context

每个 ability 的描述会 leak 到 context window

→ 用 disable model invocation: true 限制

🧠 Knowledge / Skills / Wisdom 3 维度

"Wisdom is almost impossible to obtain without actually having done the thing in the exact context where you need to do it."
维度定义获取方式
Knowledge 知道是什么 学习 / 阅读 / 看视频
Skill 知道怎么做 练习 / 实战
Wisdom 知道在真实场景怎么做 必须亲自做（不可替代）

关键洞察

Wisdom = 真正的护城河

你可能 bundle knowledge + skill → 但 wisdom 必须自己拿

→ 想成为像 Anthropic 工程师 = 必须去 Anthropic 才能 gain 那个 context 的 wisdom

⚖️ AX vs DX

"I think of there as being a difference between DX (developer experience) and AX (agent experience)."
维度DX（Developer Experience）AX（Agent Experience）
为谁优化 人类开发者 Agent
Senior 优势 强（知道怎么 build 好 DX） 强（DX 和 AX 有大量 overlap）
Junior 优势 弱 弱（但 enthusiasm）
改善方法 好代码 / 文档 / 测试 skills / harness / codebase 改进

关键洞察

DX 和 AX 有大量 overlap

但 AX 经常被忘（"improving codebase for better AX"）

→ Senior 强 DX = 自动 也 强 AX（同一套基础设施）

→ Junior 强 AI（AX）但需要学 DX 基础

Hiring 哲学

"Hiring great juniors has always been the goal of any company because if you find a great junior then anyone who's enthusiastic will do a better job than someone who's more experienced basically."

Enthusiasm beats experience（pure output）= 因为发展快

🚫 Bitter Lesson 反驳

Bitter Lesson（Rich Sutton）

"Whatever you do in machine learning research, compute, raw compute will just beat you every time."

Matt 的反驳

"If I try to overoptimize around a model, if I get too focused on the model, I will lose focus on the fundamentals."

50/50 原则

Engine + Harness 都重要

不能只等 model 升级

Codebase 优化 = 跟等 1 代 model 升级等效 + 省 90% cost

实操建议

Codebase 状态可用的 Model效果
好（clean / typed / tested） stupider model 即可 省 90% cost
差（legacy / 难改） 需要 SOTA model 贵 10x

🔁 Loop vs Cue（关键分类）
