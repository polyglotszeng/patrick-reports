"部署公司的成立和 FDE 的爆火，本质上都是在证明 Agent 时代，企业需要的不仅是一套 AI 工具，而是一套 AI-native 的工作流。"

💰 40 亿美元赛道

🚀 FDE 硅谷最火

  • Forward Deployment Engineer
  • "AI 落地官" / "前线部署工程师"

3 大核心论点

  1. 模型 ≠ 产品 = "大家还是要会承认，模型公司它会意识到模型本身并不是一个产品"
  2. AI-native 工作流 = "企业需要的不仅是一套 AI 工具，而是一套 AI-native 的工作流"
  3. 模型公司 + 应用公司边界模糊 = "亦友亦敌的关系变得更加地纠缠不清了"

① FDE 是什么？= 迷你 CTO + AI 落地官

"FDE 是在跟客户紧密合作，能够让 AI 应用真正跑起来的工程师，且它承担了让产品变得更好的这样一个职责。"

3 大核心职责

  1. AI 落地（deploy model into real workflow）
  2. Lesson Learned（把客户实战经验带回产品 = 改变产品）
  3. 像 Forward Deployed CTO（"你是非常全面一个人，你是要把单子搞定"）

类比

FDE = 迷你 CTO

技术全面

确保实现合理 + 测试健全

FDPM = 迷你 CEO

沟通 / 商业

需求把控 + 风险上报 + 追加销售

② Palantir = FDE 鼻祖

"这个模式或者这个概念，是由 Palantir 一开始发明，并且推广的。"

Palantir 起源背景

  • 10-15 年前军方业务
  • "不是每个厂商都能够去做一些军方的业务"
  • 客户不高兴说得很明 → 必须面对面跑到军营帐篷看数据
  • 中间涉及数据建模 / 临时 API → 招两团队解决问题
  • = Palantir FDE = 军方业务催生

关键 insight

"门槛还是比较高的这样一个业务，所以说导致他的客户对 Palantir 提供的工具还有提供的 FDE 产生了一个比较强的用户黏性"

= Palantir 模式 = 工具 + FDE 关系 = 客户黏性

③ 为什么现在 FDE 爆火？= Agent 时代需求

5 大原因

  1. AI 落地难 + 漫长 = "AI 落地这个事情就属于又难，时间又漫长"
  2. 客户不愿学 = "不是逼客户去学，而是我们了解你要什么"
  3. 模型公司 vs 应用公司边界模糊：
    • OpenAI / Anthropic 仍招最好的人训练模型
    • 但 360 行落地 ≠ 模型公司自己干
    • = 收购 / 资本合作（Tomoro 收购 / Blackstone 合资）
  4. AI Coding Agent 让 FDE 效率雪崩式提升：
    • "Claude Code 或者是这些模型，你可以知道哪里不对，哪里有提升空间"
    • "哪怕你有 5 个 10 个不同的 repo 不同的语言不同的栈，你就可以很容易地让 AI 帮你做成这个样子"
  5. Skill 蒸馏：
    • "现在 skill 本身，也是一个很好的模型，就能有一个方式可以蒸馏了"
    • "原先都是大家留在脑子里面，大家可能要花很多时间做知识转移，现在你可以把它写成 skill"
    • 30 FDE 做完项目 → skill 沉淀 → 20 新 FDE 直接复用 = 雪球效应

④ FDE 实际工作流（售前 → 售中 → 售后）

Jove 实操节奏

阶段任务时长
售前 功能级 PoC 样本训练 灵活
售中 启动会 + 高层级目标 + KPI + API 验证 + 小型 PoC 2-3 天闭门
售后 用例上线（分批）+ 调试 + 灰度 + 监控 2-4 月内
监控期 客户满意度 / 电话时长 / 案例解决率（合同指标） 项目级
退出 指标达合同 → FDE 退出（"后续小修小补，别的同事没那么 AI 专家的方式来去做"） 2-4 月

3 大关键 insight

  • 🔴 "一旦符合合同，其实没有必要继续把人放在里面"
  • 🟢 "FDE 本身不是以小时收费的" = 不操心 ROI = 全力以赴
  • 🟡 "我们倒贴一些人进去，他尝到甜头之后，就像送货上门" 门口试穿" = AI 时代 节奏非常快

⑥ FDE 招聘硬要求（Jove 标准）
