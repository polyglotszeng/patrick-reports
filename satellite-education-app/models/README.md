# 3D Model Directory

This directory holds GLB files for the Atlas tab 3D preview. The app
auto-loads a GLB if one exists; otherwise falls back to procedural geometry.

## File naming convention

Drop a `.glb` file named after the satellite ID (lowercased, no prefix):

| Satellite | Expected filename |
|---|---|
| ISS | `sat-iss.glb` |
| Hubble | `sat-hubble.glb` |
| JWST | `sat-jwst.glb` |
| Starlink v2 mini | `sat-slk.glb` |
| GPS Block III | `sat-gps.glb` |
| GOES-18 | `sat-goes.glb` |
| Sentinel-2 | `sat-s2.glb` |
| Landsat-9 | `sat-l9.glb` |
| NOAA-20 | `sat-n20.glb` |
| 1U CubeSat | `sat-cs1.glb` |

When a GLB is loaded, the modal shows a `🛰 GLB model` badge.
When procedural is used, it shows `⚙ procedural`.

## Generating GLB models

You can replace the procedural models with real GLB files generated by:

- **Meshy.ai** — image-to-3D or text-to-3D, free tier 10 credits/month
  - API: `https://api.meshy.ai/v1/image-to-3d` (POST multipart with image)
  - Export as GLB
- **Tripo3D** — `https://platform.tripo3d.ai/`
- **TripoSR** (Stability AI, open source) — run locally with PyTorch
- **Hunyuan3D** (Tencent, open source) — run locally
- **Luma Genie** — text-to-3D
- **Rodin** (Hexa) — image-to-3D

## Recommended workflow

1. Take a screenshot of the current procedural model (use the modal in your browser)
2. Upload the screenshot to your chosen 3D generator
3. Generate the model (5-30 min for free tiers)
4. Download as GLB
5. Drop into this directory with the correct filename
6. Refresh the Atlas tab in `mvp.html` — the GLB badge appears

## Tips

- The model should fit in a ~2 unit box (modal auto-fits camera)
- No specific texture required — flat colors work fine for the dark theme
- For real-world reference images, NASA's 3D model library and ESA's
  spacecraft galleries are public-domain sources
