03工作原理

5 分钟从零到跑起来。无需博士学位。

STEP 01
下载压缩包
从 CDN / GitHub release / S3 下载最新版压缩包
STEP 02
解压 + 引导
unzip + bash bootstrap.sh — 装 Hermes 配置到 ~/.hermes/，配 launchd 任务，加 alias
STEP 03
接入模型
填上 Claude / GPT / 智谱 API key。或者跑本地 llama.cpp。Hermes 按任务自动路由。
STEP 04
跑第一个任务
"hermes 给我搭个日报" 或 "审计我的 CF Pages" 或 "总结我的邮件" — 自然语言直接出活
STEP 05
设置定时
hermes schedule "早间 AI 摘要" --cron "0 8 * * *" — 每天早上 8 点自动跑
STEP 06
定制并发布
改 AGENTS.md，加你自己的技能，做一个细分版本。重新打包发布。永远属于你。

06常见问题

用户真正会问的问题。

真免费吗？有什么套路？

真的免费。代码 MIT 协议 — 复制、修改、卖、发布，无须署名。唯一「成本」是你自己的模型 API key（Claude 轻度使用约 $5/月，或者用本地大模型免费）。Julian 的 $67/月买的不是代码，是社区 + 便利。如果你觉得便利比成本重要，就给 Julian 付费。如果你想学、想拥有，用这个。

为什么用这个不用 Julian 的？

如果你想：零配置、不在乎月费、看重他的社区 → 用 Julian 的。如果你想：弄懂这套系统、深度定制、或者基于此做自己的产品 → 用这个。两种都合理。Hermes 内核是同一个。

这跟 Hermes Agent 是一回事吗？

对 — Patrick 的 Agent OS 是「基于」Hermes（开源 AI 智能体）打造的。它是一套预配好、策展好的安装包。Hermes 干 AI 的活；Patrick 的 pack 在它之上加了 cron 任务、记忆系统、PITFALLS、技能库。

我能卖自己的版本吗？

能。MIT 协议。做一个细分 Agent OS（给 SEO 玩家、电商、律师、医生等），想收多少收多少。Patrick 的 pack 就是你的起跑线。

能在 Windows / Linux 跑吗？

Patrick 的 pack 优先 macOS（用 launchd）。Hermes 内核是跨平台的。Linux 版本（用 systemd timers）在路线图上。Windows 还没测过。

这跟直接用 Claude Code 或 Cursor 有啥不一样？

Claude Code / Cursor 是交互式编程工具。这是一整套给 AI 智能体用的操作系统 — 它在后台跑任务、调度任务、从错误里学（PITFALLS）、跨会话记上下文。两个都该用：自动化用这个，交互编程用 Claude Code。
