✓ Patrick 担保 — Patrick 完整看过 + 5 句注解 + 5 个 cross-ref
· auto-aggregated — 自动收录但未深读（仅 entry-level metadata）

§1 两种 badge 的语义

Badge含义操作深度
✓ PatrickPatrick 完整看过 + 加 5 句注解 + 5 个 cross-ref深 (≥30 分钟投入)
· auto自动收录 / 浅扫 / 仅 metadata浅 (<5 分钟)

§2 担保流程（4 步）

步骤 1 — 发现

Patrick 从 daily 工具源（arXiv / HN / X / newsletter / vault）里看到候选 entry。

步骤 2 — 5 句注解

Patrick 写 5 句注解：

  • 这内容真正在讲什么
  • 我同意 / 不同意哪一点 + 为什么
  • 如果错了，最大可能错在哪（反向 pressure test）
  • 跟过去 100 条比，新增了什么 insight
  • 5 年后回头看，这条还会被引用吗

步骤 3 — 5 个 cross-ref

Patrick 找 5 个 cross-reference，链接到 vault 里的相关笔记 / daily-reports 历史 entry / external 长文。

步骤 4 — 担保标记

把 curated: true 字段写到 reports.json + 加 ✓ Patrick badge 进 HTML。

§3 撤回机制（如果 Patrick 看错了）

规则Patrick 一旦发现担保过的内容有事实错误 / 立场过时 / 关键信息缺失，立刻：

  • 从 reports.json 移除 curated: true
  • badge 变灰（auto）
  • 公开 +1 篇 "我为什么撤回" 说明，包含原始担保日期 / 撤回原因 / 改正建议

撤回 = 公开承认，不是隐藏。 这才是真 provenance。

§4 为什么这个 badge 难以伪造

层面为什么不能轻易复制
视觉HTML + CSS + lime 品牌色组合可以复制，但 没有任何可信度
域名真 provenance 只在 2017zyl.xyz / patrick-reports.pages.dev 上生效，第三方复制 = 来源不同
Git 历史git log 显示每条 curated:true 的精确 commit + 作者 + timestamp，无法伪造时间
Cross-ref 链5 个 cross-ref 指向 Patrick 的 vault 笔记 / 历史 entry，需要真的读过才能写出
5 句注解annotation_count = 5 是显式 metadata，复制 HTML 但 annotation 内容空洞一眼看出

§5 "5 + 5 + 5" signature

Patrick's signature · 出现在所有产出 5 + 5 + 5 · 5 条本周必读 · 每条 5 句注解 · 5 个 cross-ref · 2026-07-14

「5+5+5」是 Patrick 的 curator signature — 看到就知道是 Patrick 本人选过 + 加过注解的内容，不是自动收的。

§6 透明声明

立场Patrick 担保 ≠ 真理保证。Patrick 一人会错，但错了一定公开撤回。读者请自行 cross-check 5 句注解 + 5 个 cross-ref，自己判断。
