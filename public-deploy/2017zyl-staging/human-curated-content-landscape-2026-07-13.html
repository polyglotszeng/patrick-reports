§1概念:Human-Curated Content 是什么

1.1 一句话定义

由真实的人类策展人 (curator) 通过品味 + 判断 + 专业知识主动筛选、组织、注解、排序的内容。

1.2 3 个不可妥协特征

特征含义缺失会变成什么
Human Selection真人主动挑选,不是算法 feed→ Algorithmic Feed (抖音 / X For You)
Human Voice有 editorial perspective / stance→ Aggregator (RSS 全文搬运)
Human Organization主动排序 / 注解 / contextualize→ Raw Search (Google)

1.3 2×2 坐标

Human
Machine

编辑挑
(策展 — HC)

LLM 写

Human-curated content = 左上格子。但实际有 4 种 hybrid:

  1. 人类挑 + 人类写 — Stratechery, 传统杂志
  2. 人类挑 + LLM 写 — 未来主流:curator 给 prompt + judgment,LLM 扩写
  3. LLM 挑 + 人类校 — 漏斗 + quality gate
  4. 人类挑 + UGC 收集 — 评论区 / 投稿编辑化 → Hacker News 模式

§3未来值得做的 8 个方向 (按"Patrick 可执行性"排序)

#1

Meta-Curation — AI 内容的"人类过滤"层 已在做

  • 为什么:2026 年 LLM 内容 99% 是 noise,1% signal;但 1% 没人看到 = 0% signal
  • 怎么做:"This Week in AI-Generated" — 每周人工挑 10 篇真有价值的 AI 内容 + 注解
  • 变现:Substack / paid newsletter / Patreon
  • Patrick 现状:已在做 (daily-reports 精选项目模式)
#2 🔴 缺

Vertical Niche "Taste-maker" Marketplace

  • 为什么:Substack 已验证 "单人 newsletter = 可商业化",future = "策展人 marketplace"
  • 怎么做:N 个垂直(NFT / 量子计算 / Indie Game / Postgres 内部)各 1 个 curator,付费订阅,curator 按 reading time / trust 计费
  • 变现:Tapestry 模式 (Ben Thompson 已做:Stratechery → Dithering / Sharp Tech / Game Plan)
  • Patrick 现状:未做 (vault skill 是单人 curator,不是 marketplace)
#3 ✅ 已在做

"翻译 + 注解" 跨语言策展

  • 为什么:中文圈跟英文圈信息差 6-12 个月 + 文化转译损失
  • 怎么做:不只是翻译,是"挑选 + 翻译 + 注解 + context binding"
  • 变现:付费 newsletter + 课程 + 公网 HTML
  • Patrick 现状:已在做 (daily-reports + video-to-public-deploy 双语版)
#4 🔴 缺

"我替你看" Slow Subscription

  • 为什么:algorithm 把所有人赶到 dopamine loop,人类策展的"反 algorithm 慢节奏"反而 premium
  • 怎么做:"1 月 1 本" "1 季 1 张唱片" "1 年 10 部电影",印刷 + 邮寄 + curator handwritten note
  • 变现:Stripe / direct pay + 周边产品
  • Patrick 现状:未做 (Patrick 节奏是 fast / daily)
#5 🟡 半做

Skill / 学习策展 ("你应该学什么" 比 "给你内容" 值钱 10x)

  • 为什么:Coursera / Udemy 内容爆炸,真正稀缺的是 "学什么 + 怎么学"
  • 怎么做:domain expert + LLM 协作,产出"领域学习路径 + 资源精选 + 节奏建议"
  • 变现:课程 + coaching + 顾问
  • Patrick 现状:半做 (distributed-research-playbook + multi-source-data-fetch-dedup, 但没 "该学什么" 的 curator voice)
#6 ✅ 已在做

Cross-domain "思想家策展"

  • 为什么:单一作者内容过剩,但 cross-domain synthesis 极度稀缺
  • 怎么做:让 30-100 个思想家围一张桌子,对一个 topic 各自发言(curator 写)
  • 变现:Premium HTML dashboard / 课程 / 顾问
  • Patrick 现状:已在做 (multi-perspective-salon skill)
#7 ✅ 已在做

本地化 / 文化传承策展 (中文世界最稀缺)

  • 为什么:中文 AI 内容高度重复 + 翻译,真正的"中文世界独有的策展人视角"是稀缺品
  • 怎么做:把中文圈优秀内容 + 英文圈优秀内容双向搬运,加 Patrick 自己的注解 / 立场 / 反例
  • 变现:公众号 + 知乎 + 公网 HTML
  • Patrick 现状:已在做 (vault-to-public-pipeline 精神)
#8 🔴 缺

Trust-Mark / Provenance Layer

  • 为什么:Deepfake + AI 内容泛滥,provenance = 比 curation 还稀缺
  • 怎么做:给每条内容加 "这个 curator 担保 / 这个源头可追溯" 标记
  • 变现:B2B API (CMS / platform license) + premium verification
  • Patrick 现状:未做 (没有正式 trust mark 体系)

§7策展人 Prompt Toolkit (10 prompts + 5 frameworks)

7.1 10 个策展 prompts

挑选 prompts

1. "用 30 秒告诉一个 5 岁小孩,这条内容为什么值得看" — Patrick 实战真值测试
2. "如果这条内容我只能转给 1 个人,我会转给谁,为什么" — 重要性 + relevance 测试
3. "如果 5 年后回头看,这条还会被引用吗" — 时效性 vs 永久性
4. "这条跟过去 100 条比,新增了什么 insight,而不是重复" — 增量价值

组织 prompts

5. "这 5 条如果只保留 1 条,留哪条,为什么" — 优先级测试
6. "这 5 条有什么共同模式,能提炼成什么框架" — 抽象层
7. "这 5 条如果读者只能记 1 句话,该是哪句" — TL;DR 提炼

注解 prompts

8. "我自己的立场是什么,我同意 / 不同意哪一点,为什么" — 立场注入
9. "如果这条是错的,最大可能是哪里错" — 反向 pressure test
10. "这条的内容 + 我的注解 = 什么新东西,原文没说但我说出来了" — 增量洞察
