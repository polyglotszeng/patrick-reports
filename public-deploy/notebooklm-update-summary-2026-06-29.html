三、3 大升级 (同主题 Julian Goldie 笔记 notebooklm-ai-agent-2026-06-09 早 17 天已铺过底)

🚀 升级 1 — 新底座: Gemini 3.5 + Antigravity (Google internal eval 65%/70%/78%)

[01:11] 时间窗: 1:11 – 1:27 ([0:71-0:104] 段 = 1:11-1:43). NotebookLM 底层模型换装 Gemini 3.5 + Google 自研 Antigravity 系统; 后者带来可见的 step-by-step reasoning — 你能看到它怎么走到结论, 便于在产出前抓 drift. Google's own internal benchmarks (5 个核心评估类目):

  • 综合胜出率 ~65% ([1:43] 时段).
  • 大文档分析胜出率 ~70% ([1:43] 时段).
  • 深度网研胜出率 ~78% ([1:43] 时段).

[1:34] "And Google backed this up with their own internal benchmarks." — 注意: 这些是 Google 内部 A/B 对照旧版, 不是 LMSys / 第三方盲测。

☁️ 升级 2 — 专属沙箱云电脑 + 100+ 内置 skills

[02:11] 时间窗: 1:51 – 2:49 ([2:11-3:09] 时段). 每个 notebook 默认自带 一台 sandboxed 云电脑, 可写并执行代码. 自带 100+ 内置 skills, 数据清洗、统计、画图、文档分析、工作流自动化开箱即用. 强调「没有 plugins 要装, 不需 setup」([2:45] 时段).

与 Patrick 现有栈的对照:

  • 不同于 Hermes CLI 直接读本地文件系统, NotebookLM 的云电脑是隔离 sandbox.
  • 不同于 Claude Code / Codex 类 coding agent, NotebookLM 走「研究 + 文档产出」路线, 不是写代码 / 操作电脑.

📦 升级 3 — 零源起始 + 可下载交付物

[03:21] 时间窗: 3:21 – 5:33 ([3:21-5:33] 时段). 两个 coupled 改动:

  • (a) 可下载交付物: PDF (含 charts) · Excel · PowerPoint · CSV · JSON · Markdown · TXT · Word · 图片用 Google Nano Banana 模型生成.
  • (b) [5:42] zero source 起始: 打开空 notebook, 输入问题, NotebookLM 自动上网找源 — 列出候选供你 approve / skip, 然后基于已批准源建立完整 notebook.

主播说: "The entire front end of the research process is now handled automatically." ([6:32] 时段). 类比: 概念上类似 Perplexity (web RAG) + Notion AI (产出文档) 的杂交. 与 distributed-research-playbook + graphify 思路一致.

架构层观察: NotebookLM 的新版本 → 典型「垂直化 agent」模式 — 用 RAG 锁领域, 用 sandbox 锁执行环境, 用 file builder 锁输出. Patrick 的栈更通用 (Hermes CLI + skills 系统), NotebookLM 更约束性但开箱即用.

五、反方 / 局限

1. 付费门槛: [8:08] 「这些都是为 Google AI Ultra 订阅者 + Workspace AI Ultra 访问者准备的」, 普通账号不立刻有 ([8:08-8:53] 时段).

2. "garbage in, garbage out" 仍存在: [8:31] 「Before you send any report or deck to a client or partner, give it a real look.」输出仍需人审.

3. 零源模式的精度: [8:49] 「be precise about your topic and your goal, and it'll do a much better job.」— 模糊 prompt = 垃圾结果.

4. marketing 风险: 主播 Nick Ponte 本人是 AI 卖课人, 「the last one is genuinely wild」等措辞别照单全收, 应到 Google AI 博客原文核 65/70/78 数字.

5. 生态锁定: 数据 / 源 / 产物都默认留在 Google 云, 不像 Claude / Hermes 那么便于本地化导出.

6. 不是写代码的 agent: NotebookLM 跑代码限于 sandbox 内 (分析、可视化), 不会帮你操作电脑 / 写应用; 这是 Manus / Claude Code 的赛道, 非同一.
