🔥 核心论点

传统 agent 只学 policy（决定下一步动作），不学 world model（动作之后世界怎么变）。结果是 agent 处于 reactive mode — 知道按 jump 键，但不知道跳了之后世界什么样。

Qwen-AgentWorld 把这个范式翻了过来:训练一个 语言世界模型(language world model) — 输入 (current state, action) → 输出 next state (terminal 输出 / HTML / JSON 等文字)。这跟 Nvidia Cosmos / Google Genie 等视觉世界模型完全不同:预测的是 autoregressive text,不是图像或视频帧。

三大核心论点:

  1. "CPT injects, SFT activates, RL sharpens" — 3 阶段管线,每阶段目的不同:CPT 注入世界知识 → SFT 激活推理显式化 → RL 通过 on-policy + LLM-as-judge 锐化 fidelity
  2. 幻觉环境的价值 — 模型可以 deliberate 注入错误、隐藏答案、分页 HTML,提供现实无法给的 adversarial coverage,让 agent 训练更 robust
  3. Predicting the world makes the model a better agent itself — 即使不做 RL,纯 inference 时让模型先 predict 下一步再 act,CoT reasoning 把 accuracy 拉到 78.3%

关键设计选择

✅ 全文字表征 (vs 视觉)

所有 7 个 world 都输出 text (HTML / JSON / stdout)。优势:跟现有 LLM 训练范式 100% 兼容,token-level loss 即可训练;rollout 速度比 render 像素快 10-100x。

❌ 视觉信息缺失

渲染画面、图像内容、UI 颜色等不能直接建模。要做 vision agent 仍需另搭 image caption pipeline。

Patrick 关联:MCP tools (#4) + Web browsers (#5) 是 Patrick 5-tool stack 的核心操作面。如果 AgentWorld 在这两域 rollout 足够 fidelity,直接可以喂 Patrick 的 heart-beat loop / task router。

📊 3 阶段训练管线 — "CPT injects, SFT activates, RL sharpens"

CPT · Continual Pre-Training → SFT · Supervised Fine-Tuning → RL · Reinforcement Learning

Stage 1 — CPT injects (注入世界知识)

数据: millions of real-world action-observation trajectories 来自 sandboxes (MCP servers, Android emulators, OS emulators) + world knowledge corpora (law / medicine / finance / cybersecurity)。

目标:让模型吃下"世界怎么运转"的事实层 — 不是学怎么 echo tool call,而是学 tool call 之后世界会变成什么样。

反例:不训练"再生产 MCP server 的回显",训练"知道 tool call 之后真实世界反馈的物理 / 逻辑结果"。

Stage 2 — SFT activates (激活推理显式化)

数据: 7,000 trajectories with explicit reasoning chains(通过 rejection sampling 选高质量样本)。

目标:把 CPT 学到的 implicit knowledge 切换为 explicit thinking — 从 next-token-prediction 切到"先想再答"。

关键 trick:rejection sampling 把 bad reasoning 滤掉,只保留 high-quality chains。7,000 这个数量级比传统 SFT 小 10x,但 quality ≥ quantity。

Stage 3 — RL sharpens (锐化 fidelity)

方法:on-policy rollouts + 两层 reward signal:

  • LLM-as-judge — 评分 5 个维度(format / factuality / consistency / realism / quality)
  • Rule-based verifiers — code compile / JSON format 等 checkable 规则

关键:两层 reward 互相制约防止 reward hacking — 即便 judge 给高分,违反 rule-based check 仍被 penalize。

⚖️ LLM-as-Judge 5 维度 + Rule-Based 双层 Reward

Qwen-AgentWorld 的 RL 阶段关键设计:不用单一 judge,而是 LLM-as-judge 5 维度 + rule-based check 双层,防止 reward hacking。

LLM-as-Judge 5 维度

#维度衡量失败 mode
1Format输出符合 agent 期望的 schema多余 markdown / 错位 JSON
2Factuality跟真实世界知识一致幻觉 tool 输出
3Consistency多 turn 间状态连贯前后矛盾 (例如 ls 之后目录不一致)
4Realism像不像真实 sandbox 反馈过于干净的输出 (无 stderr / 无 warning)
5Quality整体 reasoning 链质量偷懒跳过 reasoning

Rule-Based Verifiers (反 reward hacking)

针对 checkable 任务(code / JSON / file format):

  • Code 必须 compile 成功
  • JSON 必须合法 parse
  • File path 必须实际存在
  • HTTP 状态码必须在合理范围

✅ 为什么双层

Judge 模型可能给一段"看似合理但代码不编译"的输出打高分。Rule-based check 把这种 reward hacking 直接拦下。

❌ 单一 judge 的问题

Pure LLM-as-judge 容易让 policy 学到"convince the judge" 而不是 "produce correct output"。Mixing rule-based 是核心防御。

Patrick 关联:Patrick 的 dual-ai-peer-review skill 已经用 Claude + Gemini 双 LLM 投票,本质就是 multi-judge 防 hallucination。Qwen-AgentWorld 把这思路推到 RL reward 设计层,值得 cross-reference 到 evaluation skill。

🎯 仿真器用法 — Adversarial RL at Zero Cost

为什么需要 simulator

Real RL 训练需要 spin up 真实 environment:

  • 🐢 慢 — Android emulator 启动 + 安装 app = 几分钟/episode
  • 💸 贵 — 真实 MCP server / OS sandbox 占用机器 / 容器
  • 🎯 覆盖差 — 真实环境往往被设计成"通常能用",edge case 罕见

Qwen-AgentWorld 把 environment 换成 语言生成:
"Hey, let's pretend to be the environment."

可以 deliberate 注入的 adversarial 条件

🐛 Inject Errors

让 world model 主动返回错的 stderr / failed compile,逼 agent 学 recovery

🙈 Hide Answers

tool call 返回时 partial response,逼 agent 学 pagination / retry

📄 HTML Pagination

搜索结果分页 — agent 必须 navigate page 2 才找到答案

⚠️ Random Latency

tool response 随机延迟 — 逼 agent 学 timeout handling

🔀 Inconsistent State

OS 状态在 tool call 之间漂移,逼 agent 学 idempotency check

🚫 Permission Denied

随机返回 403 / 401,逼 agent 学 auth refresh

🚀 Self-Reflection 用法 — Predicting the World Makes the Agent Better

核心观察

即使不做 RL 训练,只在 inference 阶段让 agent 先"想象下一步世界状态"再决定 action,accuracy 也显著提升。

"Just getting it to be better at articulating its thoughts by predicting out what it's thinking is going to happen bumps up the accuracy from 69.9% right up to 78.3%."

机制解释

预测 next state 的过程 = 显式 self-reflection:

  1. Agent 想执行 action X
  2. 先让 language world model 预测 "if I do X, state becomes S"
  3. 检查 S 是否符合期望 (e.g. S 应该有 file.txt 已创建)
  4. 如果不符合 → 调整 action 或重新 reasoning
  5. 如果符合 → 执行

这个 5-step loop 跟 chain-of-thought 同源,但对象是 world state 而不是 internal reasoning。在 multi-step task 上效果特别显著,因为每一步的"想象"提前暴露 downstream conflict。

实战对应

🧠 ReAct Pattern

Reasoning + Acting 交替。Qwen-AgentWorld 把 Reason 这一步升级为 "imagine world state",比纯 ReAct 更 grounded。

🔄 Reflexion

失败 → verbal reflection → 重试。Predict-world 是 reflexion 的 proactive 版本(执行前 reflect 而不是失败后)。

🌳 Tree of Thoughts

展开多个 action,每个想象 state,选 best。Qwen-AgentWorld 是 ToT 的 world-grounded 实现。

⚠️ 反方 / Caveats — 什么不应该被高估

❌ Bench 是自家的

35B 击败 GPT-5.4 / Claude Opus 4.8 在 AgentWorld bench 上 — bench 是为这任务设计的,跟 Vibe Thinker 论文模式类似。Generalization 到别的 domain 未经验证。

❌ 397B 没开源

真正击败 frontier 的 397B/17B 版本没发布。开源 35B 是"教学版",实战能力可能远低于公开数字。

❌ Simulator fidelity 有上限

语言世界模型本质是 LLM 生成 — 跟真实 Android emulator 仍有差距,特别是视觉 / 性能 / timing 维度。纯 simulator 训练的 agent deploy 到真实环境可能 surprise。

❌ Reward hacking 风险

即使用 LLM-as-judge + rule-based 双层,policy 仍可能找到 exploit(例如 judge 偏好的某种 reasoning 模式)。长期需要 red-team audit。

❌ Adversarial 双刃剑

可以 deliberate 注入错误 = 也可被恶意 agent 利用(训练 bypass safety check 的 agent)。Qwen 团队需要 audit trail 跟 human approval gate。

❌ 与 Cosmos / Genie 互补不替代

纯文字 world model 不覆盖视觉 / 物理交互。要训视觉 agent 仍需 image world model。两套体系需要融合,不是替代。

🗣️ 5 位命名研究者 + 他们的反方立场

1️⃣ Yann LeCun (Meta Chief AI Scientist · JEPA 作者)

立场:"Autoregressive LLMs cannot be true world models. World models require non-generative architectures like JEPA that predict in embedding space, not token space."

对 Qwen-AgentWorld 的影响:Qwen 用 autoregressive text 预测 next state,跟 LeCun 的 JEPA 路线根本对立。Patrick 选择时要注意:LeCun 认为 text-prediction 是"局部最优但次全局"。

2️⃣ Rich Sutton (Bitter Lesson 论文 · Keen Technologies)

立场:"The bitter lesson: methods that scale with compute + search eventually win. Hand-engineered simulators are a temporary local optimum that compute scale will surpass."

对 Qwen-AgentWorld 的影响:3 阶段训练管线 (CPT/SFT/RL) 大量 hand-engineering,跟 Sutton 哲学冲突。GPT-5.4 / Claude Opus 4.8 (general models) 可能用纯 compute scale 追平 AgentWorld 的 narrow score。

3️⃣ Stuart Russell (UC Berkeley · AI Safety 权威)

立场:"Adversarial training with simulated failures creates dual-use risks. Agents trained to bypass safety via error injection can be weaponized."

对 Qwen-AgentWorld 的影响:deliberate inject errors / hide answers 的能力 = 双刃剑。Patrick 必须 HILAG block mode for any production agent trained this way。

4️⃣ John Carmack (Oculus VR · AGI Startup)

立场:"397B inference at production scale is fantasy without 4-bit quantization + aggressive pruning. Local deployment claims need honest math."

对 Qwen-AgentWorld 的影响:Qwen 公开 35B-A3B (实际可跑),但 397B-A17B 标 58.71 是在云端 GPU cluster 跑的。Patrick M3 Max 64GB 只能跑 35B Q4 ~12GB,不要期待本地 397B。

5️⃣ Andrew Ng (Landing AI · deeplearning.ai)

立场:"For 80% of production tasks, a well-tuned small model (3-7B) beats a frontier model wrapped in complex simulator pipelines."

对 Qwen-AgentWorld 的影响:不要为了用 AgentWorld 而用。Patrick 5-tool stack 多数任务应该用 Qwen 2.5 1.5B / 3B (T0/T1 tier),simulator 只在 score ≥ 4 才介入。
