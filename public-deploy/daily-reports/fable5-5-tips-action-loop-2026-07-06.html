🏆 Step 3 实操指南 (立刻可用)

推荐: 加 Verbosity Guidelines 到 Hermes system prompt

基于实测, Ponytail-equivalent guidelines 在 MiniMax-M3 上 = 40% cost 节省 + 1.5x faster output.

# ~/.hermes/config.yaml 加 system_prompt_append
agent:
  system_prompt_append: |
    ## Output verbosity guidelines (Ponytail-equivalent)
    - Skip unnecessary prose ("Sure!", "Here is the code:", "Let me know if...")
    - Skip verbose explanations of what the code does (the code itself is the explanation)
    - Use tight, dense code with minimal comments
    - Prefer single-line expressions over multi-line when readable
    - Test cases should be terse: assertEqual(f([1]), [1])
    - No emojis, no padding, no marketing-speak
    - Target: same functionality, ~30-40% fewer output tokens

⚠️ 4 步实测 caveat

Task 复杂度 bias: 实测 task 是 50-line simple function (1 file, 1 class). 结论不适用 multi-step complex task (e.g. refactor 1000-line codebase, build 5-file feature). 那种场景:
  • Step 2 Architect 可能省 30-50% (因为 plan 避免大量 false starts)
  • Step 4 Advisor 可能省 10-20% (因为 second opinion 避免 rework)
  • Step 3 Verbosity 仍然 30-40% (跟 task 复杂度无关)
Rule of thumb: 1 call task = Step 3 only. 2-5 call task = Step 3 + Step 2. 5+ call task = Step 3 + Step 2 + Step 4.
