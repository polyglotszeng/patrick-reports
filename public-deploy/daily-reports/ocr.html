⚡ 概述 — 这期视频讲什么

百度 PaddleOCR 团队 (8 万+ 星老牌 OCR 仓库) 与 DeepSeek 的端到端 OCR 思路碰撞,推出 Unlimited OCR。核心 trick 是把 DeepSeek OCR 的多头注意力 (MHA) 替换为滑动窗口注意力 (R-SWA, Rectangular Sliding Window Attention),让 OCR 模型可以一次吃下一整本书的 PDF 并连续输出,告别"逐页扫描"时代。

关键判断: 5 天破万星背后的真正共识不是 Unlimited OCR 本身多强,而是大家认 R-SWA 这条架构路线 — 这是 OCR 领域的技术拐点信号。

📐 核心论点 (TL;DR)

百度 PaddleOCR 团队 (8 万+ 星的老牌 OCR 仓库) 与 DeepSeek 的端到端 OCR 思路碰撞,推出了 Unlimited OCR — 通过把 DeepSeek OCR 的多头注意力 (MHA) 替换为 滑动窗口注意力 (R-SWA),让 OCR 模型可以一次吃下一整本书的 PDF 并连续输出,告别"逐页扫描"时代。

为什么是"新篇章": 不是单纯指标涨点,而是底层架构变革 — R-SWA 把"能不能连续看多页"这个底座问题解决了,后续跨页表格合并、跨页上下文割裂、prefill pool 拉长到 128K+,都是堆数据和工程优化的事。这是一条技术路线的拐点,不是单点升级。

详细论点 (6 段)

1. OCR 三段式流水线 vs 端到端大模型

  • 传统流水线 (PaddleOCR 经典款): 文字框检测 → 框内文字识别 → 版面拼合。优势是工业级稳定,但任一环节出错,后续全错(误差累积)。
  • VLM 路线 (Qwen-VL / GPT-4V 等): 端到端好,但贵且慢 — "大炮打蚊子"。
  • DeepSeek OCR (3B 参数): 第一个真正为 OCR 优化的端到端大模型,在文档解析 Benchmark 上能跟 235B 掰手腕。
  • PaddleOCR-VL (0.9B!): 比 3B 还小,效果依然能打 — 参数效率证明端到端路线成立。
  • 共同短板: 文档越大越慢,必须一页一页扫,跨页上下文割裂。

2. Unlimited OCR = R-SWA

  • 核心 trick: 把 DeepSeek OCR 的多头注意力 (MHA) 改成滑动窗口注意力 (R-SWA, Rectangular Sliding Window Attention)。
  • 人话类比: 像人抄书 — 书一直摆在眼前,但脑子里只需要记"刚才抄到哪"那一小段上下文。
  • 效果: 模型可以一次吃进一整本书的 PDF,一口气输出。30 页 / 50 页 / 100 页都不卡。
  • 类比大模型本身: 这跟大模型"上下文窗口"的演进逻辑一样 — 从 4K → 32K → 128K → 无限,但 OCR 这边是把它做成了架构级默认,而不是事后优化。

3. 实测数据 (up 主自己跑,基准代码 Codex 生成)

测试设置: 30 页英文论文 PDF (表格/公式/版面结构复杂) + 30 页中文剧本杀手册 (纯长文本)。指标: Edit Distance / Distinct-n / Line Recall / Number Recall。基准: DeepSeek OCR 单页扫描。

结果 (惊喜):

  • Distinct 35 / Distinct 20 几乎满分 → 幻觉极少,大模型最严重的复读问题在 OCR 场景被 R-SWA 抑制了
  • 论文 Edit Distance 看起来不高: 不是"读不出来",而是 OCR 输出和 PDF 文本层结构不一样(论文有表格公式),完全可用
  • Line Recall 和 Number Recall (正文召回) 比 DeepSeek OCR 还高 ✓
  • 中文长文本 (小说/手册): 不论一页页扫 vs 一次扫 20/30 页,质量差距极小,字级准确率非常高
  • 32K 上下文跑满,性能不衰减 ✓ (跟官方技术报告一致)

4. 当前版本限制 (up 主亲自踩坑)

  • ⚠️ 跨页表格合并还没天然支持: 模型能连续读多页,但还不理解"上页表格没结束 → 下页要继续合并"。架构底座已就位,合并靠堆训练数据。
  • ⚠️ 同时扫 10 页会有轻微质量损失 (官方报告) — up 主不太信,实测后差距确实小,但有
  • ⚠️ NVIDIA 50 系显卡 (Blackwell, sm120) 暂不支持 Flash Attention 3: 回退到 FlashInfer → R-SWA 处理有问题,up 主差点以为是模型能力差去推特骂人,最后是 Codex 打了点小 patch 救场。50 系用户稍微等官方 PR 修正。
  • ⏱ 速度: up 主卡不支持 FA3,大部分场景 ~300+ tokens/s,还能优化

5. 未来路线 (技术报告明示)

  • 短期: 上下文拉到 128K
  • 长期: 论文提到的 prefill pool — 真正接近"无限 OCR"
  • 横向扩展: R-SWA 不只 OCR — 技术报告明示会扩展到 ASR / 翻译等长序列任务

6. 为什么 PaddleOCR 团队要"认真看"端到端这条路?

PaddleOCR 是 8 万+ 星的 OCR 江湖老大哥,长期统治工业流水线。但:

  • DeepSeek OCR 证明了端到端路线是对的 (3B 打 235B)
  • Unlimited OCR 把这条路延展到新领域 (连续多页)
  • 从此逐页扫描的时代过去了,以后是一口气吃下一本书

🔄 OCR 三段式演进

阶段 1 — 传统流水线

PaddleOCR 经典款: 文字框检测 → 框内文字识别 → 版面拼合。

优势: 工业级稳定,8 万+ 星统治地位。

弱点: 任一环节出错,后续全错(误差累积,无回滚机制)。

阶段 2 — VLM 端到端

Qwen-VL / GPT-4V 等: 用通用视觉语言大模型直接 OCR。

优势: 端到端,无误差累积。

弱点: 贵且慢 — "大炮打蚊子",每个 token 都要钱。

阶段 2.5 — DeepSeek OCR (3B)

专门为 OCR 优化的端到端大模型。

效果: 在文档解析 Benchmark 上能跟 235B 掰手腕。

意义: 证明 OCR 可以端到端 + 小模型。

阶段 2.7 — PaddleOCR-VL (0.9B)

比 3B 还小,效果依然能打。

意义: 参数效率证明端到端路线成立。

阶段 3 — Unlimited OCR (R-SWA)

本次拐点: MHA → R-SWA,一次吃一整本书。

5 天破万星 = 技术路线被社区认可。

意义: 架构底座已就位,后续靠堆数据 + 工程优化。

🪟 R-SWA 架构 (滑动窗口注意力)

核心 trick

把 DeepSeek OCR 的多头注意力 (MHA) 替换为滑动窗口注意力 (R-SWA, Rectangular Sliding Window Attention)。

人话类比 — 像人抄书

"书一直都在眼前对吧, 但是人脑子里只需要记得刚刚抄到哪的那一小段就行。" — 小天fotos, 1:32-2:03

技术对应

  • 全局可见: 整本书的 PDF token 都在 attention 范围内
  • 局部计算: 每次只计算当前 token + 邻近窗口内的 attention 权重
  • 复杂度: O(n) 而非 O(n²),内存友好
  • 类比大模型上下文窗口演进: 4K → 32K → 128K → 无限,但 OCR 是架构级默认,不是事后优化

关键效果

  • ✅ 一次吃进整本书 PDF,一口气输出
  • ✅ 30 / 50 / 100 页都不卡
  • ✅ 32K 上下文跑满,性能不衰减
  • ✅ Distinct 35/20 几乎满分 — 大模型复读问题被抑制

⚠️ 50 系显卡用户必看

NVIDIA Blackwell (sm120) 不支持 Flash Attention 3,回退到 FlashInfer 后 R-SWA 处理有问题。up 主实测踩坑 → 差点以为是模型能力差 → 最后 Codex 打 patch 救场。50 系用户预期会有 2-4 周等官方 PR 修复。

💡 5 条关键 Takeaway

1. OCR 三段式演进 + Unlimited OCR 站在第三个拐点

从流水线 → 端到端大模型 → 滑动窗口多页处理,Unlimited OCR 是当下拐点。

2. R-SWA 不是 trick,是架构变革

人抄书的比喻很贴切 — 全局可见 + 局部记忆,跟大模型本身的上下文窗口演进同源。

3. 实测数据 > 官方 benchmark

up 主用 Codex 生成基准代码 + 多模型 review,验证 distinct/recall 都比 DeepSeek OCR 单页模式好 — 少有的"up 主实测反向验证官方报告"案例。

4. 跨页合并是下一个战场

架构底座已就位,数据 + 微调就能补齐 — 值得长期跟踪 prefill pool 进展。

5. 架构红利外溢 — 长序列建模范式迁移信号

R-SWA 会扩展到 ASR / 翻译等长序列任务 — OCR 这次拐点不是孤例,是整个长序列建模范式的迁移信号。

🔴 反方观点 / Bull-Bear 分析

🐂 Bull Case (技术拐点信号)

  • 5 天破万星 + PaddleOCR 8 万+ 星老大哥站台 = 社区认路线
  • R-SWA 架构底座已就位 → 后续是堆数据问题
  • 架构红利外溢到 ASR / 翻译 = 长期 LTV 高
  • 实测反向验证官方报告 = 少有的硬证据

🐻 Bear Case (冷静质疑)

  • 跨页表格合并还没天然支持 — 业务真用跨页表格的人,会非常失望 (up 主自己都踩到了)
  • 当前版本 Edit Distance 看起来不高 — 复杂版面(论文) 真正落地的工程质量还要看
  • 50 系 Blackwell 不兼容 FA3 — 高端硬件用户反而被卡住,生态分裂
  • FlashInfer fallback R-SWA 有 bug — 至少 2-4 周修复期,期间 demo 体验差
  • VLM 已经够好 — Qwen-VL / GPT-4V 在通用场景已经够用,R-SWA 多页的优势是 niche (长文档/书籍)
  • Wikipedia / arXiv 已经在做长上下文 LLM — 不是新 idea,是 LLM 主线技术外溢到 OCR

⚖️ 我的判断

短期 (6 个月): 不值得现在迁移生产,观望 prefill pool + 跨页合并进展。
中期 (12-18 个月): 如果做长文档/书籍/剧本/教材类 OCR 业务,Unlimited OCR 是必试路线。
长期: R-SWA 迁移到 ASR / 翻译比 OCR 本身更有想象力。
