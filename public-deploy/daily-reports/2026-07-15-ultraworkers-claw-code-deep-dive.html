02它到底在做什么What it actually is

两件容易踩的坑,USAGE 里专门写:Two pitfalls USAGE.md calls out explicitly:

坑 1 — 401 误诊Pitfall 1 — 401 misdiagnosis

把 sk-ant-... 粘贴到 ANTHROPIC_AUTH_TOKEN 槽里,Anthropic 后端会返 401 Invalid bearer token(sk-ant 键被 reject 走 Bearer header)。正确 = sk-ant-* 必须放 ANTHROPIC_API_KEY(走 x-api-key header)。最近 claw 的报错信息会主动检测这个 shape 并加 hint。Pasting an sk-ant-... key into ANTHROPIC_AUTH_TOKEN makes Anthropic return 401 Invalid bearer token (the sk-ant keys are rejected over the Bearer header). The correct slot is ANTHROPIC_API_KEY (which uses the x-api-key header). Recent claw builds detect this exact shape and append a hint to the error.

坑 2 — provider prefix 错乱Pitfall 2 — provider prefix confusion

当你已经有 OPENAI_API_KEY、XAI_API_KEY、DASHSCOPE_API_KEY 之一,但想跑 Anthropic:别忘了给模型名加 openai/、grok、qwen-plus 等 prefix,前缀路由胜于 env 嗅探。错误信息会报 "missing Anthropic credentials" 同时把检测到的 env var 名附上。When you already have OPENAI_API_KEY / XAI_API_KEY / DASHSCOPE_API_KEY exported but want to run Anthropic: don't forget the openai/ / grok / qwen-plus model-name prefix. The prefix router wins over env sniffing. The error message reports "missing Anthropic credentials" while naming the env var it detected.
