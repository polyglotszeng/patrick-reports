"造卫星 = 7 大子系统的工程权衡 + 倒着设计 (从任务 → 载荷 → 子系统)。工程师的核心能力不是精通某子系统，而是在受限预算/质量/功率下做系统级 trade-off。"

⭐ Power Budget 计算法

Gopal — 公式
P_orbit_avg = P_solar × S × cos(θ) × T_sun / T_orbit − P_losses
约束：P_orbit_avg ≥ Σ(P_subsystem × duty_cycle)
否则电池每天深度放电 → 任务寿命缩短

⭐ 链路预算 7 步法

  1. EIRP = P_TX + G_antenna（卫星侧等效全向辐射功率）
  2. Path Loss = 32.45 + 20·log(f_MHz) + 20·log(d_km)（自由空间损耗）
  3. Received Power = EIRP − Path Loss + G_ground
  4. C/N = Received Power − kTB（载噪比）
  5. Eb/N0 = C/N − 10·log(R)（每比特能量噪声比）
  6. Margin = Eb/N0 − Eb/N0_required（链路裕量）
  7. 判决：Margin ≥ 3 dB = 可用；< 3 dB = 加天线/加功率

⭐ 软件开发 = 12 步流程

  1. 需求：任务定义 → 软件需求规范 (SRS)
  2. 架构：状态机 / 任务调度器 / 事件驱动
  3. 接口：与 EPS / Comms / ADCS / Payload 的数据协议
  4. 开发：C/C++ 为主，Rust 渐起
  5. 单元测试：地面硬件在环 (HIL)
  6. 集成测试：整星电测试 (FlatSat)
  7. 环境测试：热真空 / 振动 / 电磁兼容
  8. 软件验证：覆盖率 / 代码评审
  9. 回归测试：每次代码变更全测试
  10. Flight Model 冻结：发射代码 100% 锁定
  11. Safe Mode 验证：故障时只保留最低功能
  12. Watchdog 验证：死机自动重启

⑨ 「倒着设计」5 步法

"⭐ Gopal 的核心方法论：从任务开始 → 推导 → 设计。从硬件开始 = 100% 会重做。"

Step 1: 任务定义 (Mission Definition)

问自己 4 个问题

  • WHY: 为什么需要这颗卫星？商业价值？科研？教育？
  • WHAT: 要观测什么 / 测量什么 / 通信什么？
  • WHO: 客户 / 用户 / 利益相关方是谁？
  • WHEN/HOW MUCH: 任务寿命 / 预算 / 进度约束？

Step 2: 载荷驱动 (Payload-Driven)

从任务推导载荷需求

  • 要观测城市交通 → 高分辨率相机 + 红外面阵
  • 要测海洋盐度 → L 波段辐射计
  • 要验证新算法 → SDR + 高性能 FPGA
  • 要业余通联 → UHF 转发器 + 简单 FM 调制

Step 3: 子系统派生 (Subsystem Derivation)

从载荷推 EPS

载荷 X W + ADCS Y W + Comms Z W = 总功率

→ 太阳能板峰值 / 电池容量

从载荷推 Comms

载荷数据率 M Mbps

→ 频段 / 发射功率 / 天线尺寸

从载荷推 ADCS

载荷指向精度 ±0.1°

→ 反作用轮 + 星敏感器

从载荷推 Thermal

载荷工作温度 -10°C ~ +40°C

→ 加热器 / 散热器 / MLI

Step 4: 接口冻结 (Interface Freeze)

⭐ 关键里程碑

  • 所有子系统的电接口（电压 / 电流 / 时序）冻结
  • 所有子系统的机械接口（尺寸 / 安装孔位）冻结
  • 所有子系统的数据接口（协议 / 消息格式）冻结
  • 所有子系统的热接口（接触面 / 散热路径）冻结
  • 冻结后任何变更 = 全系统重新评审 (CCB)

Step 5: 测试验证 (Test & Verification)

测试金字塔（借鉴软件工程）

  • 单元测试：每个子系统的组件级测试
  • 集成测试：FlatSat 整星电测试
  • 环境测试：热真空 / 振动 / 冲击 / EMC
  • 系统测试：整星模拟在轨运行
  • 验收测试：发射前最终验证

"我先买块板子再说"

→ 99% 会重做

❌ 2. 单点故障

"OBC 死 = 全系统死"

→ 必须有 watchdog

❌ 3. 接口不冻结

"EPS 改一下，Comms 也改一下"

→ 接口频繁变 = 项目崩溃

❌ 4. 测试不足

"时间紧，测试从简"

→ 地面没测 = 太空翻车

❌ 5. 过度工程

"加 100 个功能"

→ 复杂度爆炸 = 任务失败

💬 关键金句库

⭐ Gopal — 倒着设计
"Don't start with hardware. Start with the mission. If you start with hardware, you'll redesign it 3 times. If you start with mission, you'll get it right the first time."
⭐ Gopal — 系统思维
"Engineers are not experts in one domain. Engineers make trade-offs across all domains: mass, power, cost, complexity, schedule. That's what makes an engineer."
⭐ Gopal — Payload 哲学
"90% of your design time should be on the payload. The bus is just a delivery vehicle. Most student projects get this backwards."
⭐ Gopal — 真空环境
"In space, you can't use fans. You can't use convection. The only way to get rid of heat is radiation. This single fact drives 80% of thermal design decisions."
⭐ Gopal — OBC 核心
"The OBC is the system hub. Every subsystem talks to the OBC. If the OBC fails, the satellite is dead. That's why safe mode and watchdog are non-negotiable."
⭐ Gopal — 接口冻结
"Interface freeze is the most important milestone in satellite development. After freeze, any change goes through a Change Control Board. Without freeze, projects die."
⭐ Gopal — 测试金字塔
"Test like a software engineer. Unit tests on every component. Integration tests on the FlatSat. Environmental tests. The test pyramid is the same in space."
⭐ Gopal — 工程师 vs Hacker
"Hackers build things that work for them. Engineers build things that work for the mission, within constraints, on schedule, and can be maintained by someone else."
