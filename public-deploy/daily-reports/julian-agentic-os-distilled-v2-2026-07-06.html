3. Paperclip 系统 — 多 agent 组织树 (6:24 真言)

这是 14 分钟视频里 最被低估 一段 — 看视频不能跳：

                              CEO agent (审批新 agent hire)
                                │
        ┌──────────────┬────────┴────────┬──────────────┐
   Dept Leader A  Dept Leader B  Dept Leader C  Dept Leader D
        │              │              │              │
   ┌────┴────┐    ┌────┴────┐    ┌────┴────┐    ┌────┴────┐
  Worker  Worker Worker Worker Worker Worker Worker  Worker
   A1 A2    B1 B2   C1 C2    D1 D2

关键设计：每个 agent 都是 queue + scheduler + inbox 三件套。每个 agent 的 inbox 允许 CEO agent 提问/征得人类 approval。CEO 看的是"全部完成的工件"而不是 log — 类似 OpenClaw 的 status contract。Loop engine 串到 scheduler — deadline 没满足自动 escalate 到 dept leader 一层。

对应的 Julian 视频原话 [07:13] "We've even got an inbox where my agents can actually message me and ask me questions or say 'Hey, can you approve this hire of another agent?'" — 这一句是 human-loop 已经在 product 的位置，你的 human-loop-watchdog cron 正是 Julian 已经 ship 的同义功能。
