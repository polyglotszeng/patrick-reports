🧪 Mixture of Agents 核心机制

MOA = Karpathy 2024 paper 的开源实现, Nous Research 集成到 Hermes Agent。机制 = 多独立 model 各自 submit response, 一个 aggregator model 综合。不是 MoE (单 model 内部多 experts)。

David 4 大澄清

  1. MOA ≠ MoE: MoE 是 inference 优化 (单模型内部多 experts), MOA 是 orchestration (多独立 model)。
  2. Aggregator = 最强 model: 默认 = Claude Opus 4.8 / GPT 5.5 (前沿最贵最准), 但可配。
  3. Token 多但质量高: 比单 model 多 4-5 倍 token, 但 "universe diversity" 让 aggregator 比单 model 更 robust。
  4. 故障自动 fallback: 4 个 sub-model 中 1 个 error (David 演示 GPT 5.5 第一轮 error), aggregator 仍能从其他 3 个 + 自身知识综合。

MOA is multiple independent models each submit, aggregator combines — completely orchestration. Don't confuse them." — David Ondrej [0:03:02]

🧠 Hermes Skills 体系

Hermes skills = prompt template + 路由配置 + preset 三件套, 跟 Patrick 当前的 `~/.hermes/skills/` 体系 1:1 对应。David 的核心 deliverable 是 davidra.com/skills, GitHub Actions 每天 auto-sync 最新 skills 到他的 Hermes install。

Skill 三件套 (David 演示)

  1. Goal-driven framing: "describe intent, what are you doing, what is your goal" — 不是 "task", 是 "broader achieve"。
  2. Custom CL (custom command line): 比 SL (system) 更短, 用于重复任务 (类似 Patrick 的 cron prompts)。
  3. Temperature presets: code review = 0.4 (低, predictable), brainstorm = 0.9 (高, creative), default = 0.6。

David 的 4 大 reference skills

  • GLM 5.2: 通过 Vercel Gateway 路由, low latency。
  • GPT 5.5: 通过 OpenRouter, 中等 cost, 高 quality。
  • Kimi Code: 代码任务专用, "code domain" 训练 data 强。
  • Opus 4.8: 终极 fallback, 前沿最强, 最贵。

🔀 Router / Gateway 体系

Hermes 的 Router 是真正 killer feature, 不是 chat client — 是 cost gate + orchestration hub。David 强调 2 个 gateway:

  • OpenRouter ([openrouter.ai](https://openrouter.ai)): 15-20 provider, "$5-300/月" tier, "least active time" 自动路由, subscription pass-through = 检测用户已有 ChatGPT Pro token, 简单问题走订阅零额外成本。
