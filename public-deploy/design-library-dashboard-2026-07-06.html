⚠️ PITFALLS — 8 documented

跟 viewer #pitfalls 同步. 实战踩到的真坑, 不是理论.

1. Sandbox `.dev`/`.design` TLD block

neobrutalism.dev, refero.design, interfaceindex.com. Don't mark as "down" — TLD-blocked.

2. CF wall = title-presence, NOT cf-ray

Real pages behind CF have cf-ray too. Only mark CF-WALL when body has challenge marker AND no real <title>.

3. Vercel security checkpoint

SiteInspire, vercel.com/design. x-vercel-id header + "Verifying..." body. Mark only if no real title.

4. Godly → Recent rebrand (2025)

godly.website rebranded; old links still 302. Current name is Recent.

5. Tailwind UI requires auth

200 + JS shell, no real content. PASS-SHELL, not PASS.

6. curl -w unsafe split on |

Use curl -sLi + parse last HTTP status line. header_json has pipes/semicolons.

7. Don't trust your own audit tool on first run

First probe reported 32/32 FAIL (curl parse bug), ground truth was PASS 5 min earlier. Smoke test on known-good site first.

8. PASS-SHELL ≠ FAIL

Modern sites (Tailwind UI, Magic UI, Framer) are JS-rendered. Use browser_navigate for visual verify.
