🎯 Key Takeaways

  1. 核心问题: Graph RAG 三大缺陷 — ①thematic irrelevance (噪声) ②logical inconsistency (矛盾) ③structural fragmentation (孤岛)
  2. 3 层全局记忆架构 — Ontology (schema) / Fact (实体关系) / Passage (原文证据), 三层互联 + 两向约束
  3. 3 个 agent 协同 — Extraction → Conflict Detection → Conflict Handler, 每个 agent 只做 1 件事 (LLM 不擅长多任务)
  4. Schema-Instance Alignment — 每个 fact 必须被 ontology 规则约束, 避免实体类型混乱
  5. Fact-Evidence Grounding — 每个 fact 链回原文 passage, 争议时可立即溯源
  6. 2 个桥接机制 — Type-based (同 category 实体互联) + Similarity-based (cosine 相似度弱连接)
  7. 个性化 PageRank — Google 老算法用于在线检索的图传播, ranking 关键节点
  8. Smart Reset Weights — 抑制泛型 hub (person / particle) 优先稀有高密度 passage
  9. 基准测试 — HotpotQA / WikiMultiHopQA 等, +28.26 pp vs GPT-4o-mini 零样本, 跑赢所有主流 Graph RAG

📑 Section-by-Section Summary

Part 1: Graph RAG 三大问题 (0:54-2:20) — "Why MemGraphRAG"

1. Thematic Irrelevance (噪声): 例子: 癌症医学文本, LLM 抽出 "patient prefers tea over coffee" 这种无关三元组。后果: 大量噪声淹没真正的医学证据。

2. Logical Inconsistency (矛盾): 例子: 一篇文档说 "Newton born 1643", 另一篇说 "Newton born 1645"。朴素 Graph RAG 合并两个事实 → 时间线分裂, 下游检索混乱。

3. Structural Fragmentation (孤岛): 例子: 同实体写成 "Newton" 一处, "Isaac Newton" 另一处 → 互不相连的图孤岛。后果: 多跳检索无法跨孤岛遍历完整图。

Part 2: 3 层全局记忆 + 3 Agent (2:25-5:00)

┌─────────────────────────────────────────────┐ │ Ontology Layer (schema, 抽象类型) │ │ - 频率过滤 top-K schema │ │ - 例如: country-capital / person-job │ │ - weight = 5 (高频出现) │ ├─────────────────────────────────────────────┤ │ Fact Layer (实体关系, 具体实例) │ │ - 三元组: (entity, relation, entity) │ │ - 例如: (Newton, born_in, 1643) │ │ - weight = 1 (单次出现) │ ├─────────────────────────────────────────────┤ │ Passage Layer (原文证据, 可溯源) │ │ - 保留原始文本段落 │ │ - 任何 fact 争议 → 立即返回原文 │ └─────────────────────────────────────────────┘

3 个 Agent 流水线:

  1. Extraction Agent (4:08): 把新文档分到 3 层
  2. Conflict Detection Agent (5:15): 检测 fact 层矛盾 (Newton 1643 vs 1645)
  3. Conflict Handler Agent (5:30): 看原文 passage, 判定 1 个对, 修复 fact 层

重要设计原则 (5:21): "Keep agents simple. Don't give them two jobs, only one job because LLM is not that capable."

Part 3: 3 个互联图视图 (5:50-7:10)

从 3 层记忆构建 3 个图:

  1. Semantic Ontology Graph (type-relation 抽象)
  2. Fact Graph (entity-relation-entity 实体三元组)
  3. Source Evidence Graph (passage-source 原文溯源)

关键: Schema-Instance Alignment — fact 必须被 ontology 规则约束, 防止 "Newton" 被分类为 "城市" (8:13)。

关键: Fact-Evidence Grounding — fact 数学上链到 passage, 争议时 instant 溯源 (8:22)。

Part 4: 防图孤岛桥接机制 (8:45-9:30)

  1. Type-based Bridging: 不同实体若共享 ontology 高层 category, 自动连接
  2. Similarity-based Bridging: 实体嵌入向量 cosine 相似度 > 阈值时建立弱连接

全部离线完成, 在线检索只跑 PageRank (9:35)。

Part 5: 在线检索 3 步 (9:40-12:40)

用户 query Q ↓ Step 1: Memory-Guided Retrieval (并行从 3 层拉) - Schema: 找相关类型 - Fact: 找相关实体 - Passage: 找相关原文 ↓ Step 2: Structure-Aware Node Initialization - 用 semantic relevance + structural signals 算初始 node weight ↓ Step 3: Personalized PageRank (Google 100 年前老算法) - 跨图传播, ranking 关键节点 ↓ 答案给 LLM 生成最终回答

Smart Reset Weights (11:45): 抑制泛型 hub (person, particle, light) 优先稀有高密度 passage, 防止 generic 概念淹没 specific 答案。
