🔥 核心论点

Ornith 1.0 = "self-scaffolding LLMs for agentic coding"

Deep Reinforce 团队的官方定位。Agentic coding 是当前 LLM 投入最多场景。

5 大主线

1. Self-scaffolding

模型自己写 evaluation harness。Harness 不是外部固定脚本，而是 on-the-fly 生成。

2. Agentic coding 定位

Ornith 聚焦 agentic coding，但 lessons 可 apply 到其他 domains。

3. Open weights

GPT-5.6 政府限制新闻同期，35B + 9B 两个尺寸都开放权重。

4. Qwen 3.5 基底

9B 基于 Qwen 3.5，跟 Patrick 现有 Qwythos-9B BF16 同栈。M3 Max 可直接跑。

5. Harness 复用

二次任务 (Gradio) 在 Twitter harness 之后启动，模型立刻识别"已有 scaffold"。

2️⃣ Ornith 1.0 介绍 — Self-scaffolding LLMs (1:00 - 3:00)

"Why not have a model that writes its own harness, or at least has the ability to write its own harness. Therefore, if you've got a particular use case that needs one kind of harness, it can write that on the fly." — 视频, 1:30 (核心定义)
"They actually call this self-scaffolding LLMs for agentic coding." — 视频, 2:30 (官方命名)
核心定义

Harness 不是外部固定脚本，而是模型 on-the-fly 生成的执行环境。

按 use case 动态决定: Twitter API → Twitter harness, Gradio UI → Gradio harness, 复用之前的 scaffold。

跟传统范式对比

传统: model + 外部 harness (固定 repo, 维护 n 个)

Self-scaffolding: model 写自己的 harness (动态生成, 复用)

3️⃣ Why it matters — Harness 自适应 (3:00 - 5:30)

Use case vs Harness 对比

Use case传统 harnessSelf-scaffolded
Twitter API 调用写一个 twitter_harness.py模型判断需要 → 生成
Gradio UI 构建写一个 gradio_harness.py模型复用上次 scaffold

Test framework写一个 test_harness.py模型生成
实战价值

不同 use case 不用维护 n 个 harness repo，让模型自己生成 + 复用。
