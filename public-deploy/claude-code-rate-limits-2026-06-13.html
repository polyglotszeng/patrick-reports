Root cause: Each sub-agent opens a fresh context window — that context is expensive. On Pro ($20), you cannot afford parallel sub-agents.

🚀 Tip 3: The Correct Workflow

  1. Load project context — drag & drop folder into BridgeSpace or similar tool
  2. Give specific task prompt — e.g. "review security vulnerabilities in this API"
  3. Agent completes task — stop immediately, do not continue
  4. Cmd+C / Ctrl+C — abandon the agent with the big context
  5. Launch fresh agent — for the next task with clean context
Why fresh context matters: A "massive context window" from a long-running agent consumes many more tokens per turn, AND hallucinates more. One task per agent, then done.
