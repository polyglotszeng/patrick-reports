How To Think SO CLEARLY People Assume You're A Genius 元认知/思维方法
2026-06-11 19:56 422.9K 👁 19,121 ❤
⚡ 核心框架: Systems Thinking 4 Systems + DART 诊断框架
把世界拆成清晰/复杂/复合/混乱四类系统，用 DART 诊断法快速定位，并掌握每类系统的应对协议；最关键是从内部看穿自己所在的系统，打破自我设限的故事。
📌 核心要点
  • 系统=连接部件持续产出模式，从部件到连接再到模式是核心观察顺序
  • 三类陷阱：搞错系统类型、激励机制反噬（眼镜蛇效应）、反馈延迟（吸烟致癌）
  • 四种系统应对协议：Clear→流程清单；Complicated→慢分析+找专家；Complex→小实验实时纠偏；Chaotic→先稳定再求理解
  • DART 诊断框架：Deconstruct 拆部件→Analyze 看因果关系→Recognize 见过没→Test 最小测试
  • 从内部看不清自己在哪条轨道，需要 mentor / data / time 三种外部视角来定位
🔧 框架结构
Clear System：因果可见→用 checklist 严格执行
Complicated System：因果隐藏→慢下来+找对专家分析
Complex System：因果只有事后可见→小实验持续纠偏
Chaotic System：因果断裂→立刻行动先稳定再复盘

How To Become Dangerously Self-Educated (with AI) 学习方法
2026-06-03 17:41 632.2K 👁 28,673 ❤
⚡ 核心框架: ACTOR 阅读框架 (Aim / Compress / Test / Own / Run)
用 ACTOR 五步法把阅读变成主动狩猎：带使命读、压缩到树干、找反驳、复述内化、立刻行动；AI 在每一步都是副驾而非替代品，让读书真正改变决策与行为。
📌 核心要点
  • 三个阅读神话：学习风格偏好、流畅感幻觉、AI 摘要替代阅读——都在偷走真正的成长
  • A(Aim)：开始前写一句使命，决定你来猎取什么而非让书定义你
  • C(Compress)：找到书的承重树干（核心论点），别只收集叶子（划线/金句）
  • T(Test)：主动找反驳和反例，让 AI 当 sparring partner 攻击你的解读
  • O(Own)：合上书用自己的话讲出来、连到真实经历、教给 AI 检查掌握度
  • R(Run)：把书里的概念转成 1 个决策/规则/清单/实验，落地到真实世界
🔧 框架结构
Aim — 读书前一句话使命，决定读这本的用途
Compress — 找到承重主干，让细节有意义
Test — 主动寻找反驳和反例，做压力测试
Own — 用自己的语言复述 + 关联真实场景
Run — 把概念转成可执行的决策/规则/实验

14:21Thinking isn't enough and it's not finished until it helps you build something real.→ 思考不够，它要能帮你建造真实的东西才算完成。

You’re Not Behind (Yet): Learn AI Agents in 13 Minutes AI代理与Agent
2026-05-14 13:09 1.3M 👁 28,089 ❤
⚡ 核心框架: ARR 任务判定 + Agent 四角色 + GPS 任务清晰度检查 + OODA 自适应循环
区分 prompt 与 agent 的本质差异，学会用 ARR 判定哪些任务值得交给 agent；理解 agent 的 Analyst/Planner/Operator/Auditor 四角色与 OODA 自适应循环，并用 GPS 检查把任务写清楚，否则 agent 只会更快地把错事做对。
📌 核心要点
  • Prompt vs Agent：prompt 你是陪驾教练，agent 是你雇的司机——给目的地它自己走
  • ARR 三问：任务是 Autonomous(自主) / Recurring(重复) / Reviewable(可审查) 吗？都满足才是强候选
  • Agent 四角色：Analyst(找模式) / Planner(排计划) / Operator(执行) / Auditor(质检) 围绕同一个 LLM 协同
  • OODA 循环：第一条路失败时能否 Observe→Orient→Decide→Act 重新规划——这是 agent 与 workflow 的本质区别
  • GPS 检查：Goal(一句话目标) / Proof(成功标准) / Steps(每一步可描述)，不过关别自动化
  • 窄而深 > 宽而浅：赢得市场的是做透一个 workflow 一个用户痛点的 agent
  • AI 是 decoupling machine：当产出无限便宜，判断力/品味/标准定义力反而稀缺
🔧 框架结构
ARR 任务筛选：Autonomous / Recurring / Reviewable
Agent 四角色：Analyst / Planner / Operator / Auditor
GPS 检查：Goal / Proof / Steps 任务定义三件套
OODA 自适应：失败后能否重新 Observe→Orient→Decide→Act
💬 关键金句
01:34A chatbot waits for your next prompt. An agent figures out its next move.→ 聊天机器人等你的下一句提示，agent 自己决定下一步。
06:37A workflow can follow the process. An agent can reroute it completely.→ 工作流只能跟流程走，agent 可以完全改道。
08:17An agent doesn't fix bad thinking, it formalizes it.→ agent 不会修复糟糕的思考，它只是把糟糕形式化。
12:00When intelligence becomes that cheap, judgment becomes even more expensive.→ 当智能变得极便宜，判断力反而更贵了。

把世界拆成清晰/复杂/复合/混乱四类系统，用 DART 诊断法快速定位，并掌握每类系统的应对协议；最关键是从内部看穿自己所在的系统，打破自我设限的故事。
📌 核心要点
  • 系统=连接部件持续产出模式，从部件到连接再到模式是核心观察顺序
  • 三类陷阱：搞错系统类型、激励机制反噬（眼镜蛇效应）、反馈延迟（吸烟致癌）
  • 四种系统应对协议：Clear→流程清单；Complicated→慢分析+找专家；Complex→小实验实时纠偏；Chaotic→先稳定再求理解
  • DART 诊断框架：Deconstruct 拆部件→Analyze 看因果关系→Recognize 见过没→Test 最小测试
  • 从内部看不清自己在哪条轨道，需要 mentor / data / time 三种外部视角来定位
🔧 框架结构
Clear System：因果可见→用 checklist 严格执行
Complicated System：因果隐藏→慢下来+找对专家分析
Complex System：因果只有事后可见→小实验持续纠偏
Chaotic System：因果断裂→立刻行动先稳定再复盘

⚡ 核心框架: ACTOR 阅读框架 (Aim / Compress / Test / Own / Run)
用 ACTOR 五步法把阅读变成主动狩猎：带使命读、压缩到树干、找反驳、复述内化、立刻行动；AI 在每一步都是副驾而非替代品，让读书真正改变决策与行为。
📌 核心要点
  • 三个阅读神话：学习风格偏好、流畅感幻觉、AI 摘要替代阅读——都在偷走真正的成长
  • A(Aim)：开始前写一句使命，决定你来猎取什么而非让书定义你
  • C(Compress)：找到书的承重树干（核心论点），别只收集叶子（划线/金句）
  • T(Test)：主动找反驳和反例，让 AI 当 sparring partner 攻击你的解读
  • O(Own)：合上书用自己的话讲出来、连到真实经历、教给 AI 检查掌握度
  • R(Run)：把书里的概念转成 1 个决策/规则/清单/实验，落地到真实世界
🔧 框架结构
Aim — 读书前一句话使命，决定读这本的用途
Compress — 找到承重主干，让细节有意义
Test — 主动寻找反驳和反例，做压力测试
Own — 用自己的语言复述 + 关联真实场景
Run — 把概念转成可执行的决策/规则/实验
