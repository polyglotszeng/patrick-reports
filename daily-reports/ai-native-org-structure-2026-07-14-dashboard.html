为什么 "组织架构" 在 AI 时代是头号议题

软件以前能 "放大个人的腿脚"。AI 让任何白领个人能 "放大一个人的整个认知带宽"。 这意味着 公司价值不再来自 "人多", 而来自 "个人产出系数 × 人数"; 而系数的天花板被 AI 拉高后, 多一倍人 ≠ 多一倍价值。 所以组织架构的核心问题从 "如何管 1000 人" 变成 "如何让 5-10 个高杠杆个人 + 一组 AI 协作 + 一群 AI agent 做出 1000 个人的活"。

📐 软件变革三层

  • 1990s: 软件放大 "腿脚" — spreadsheet, ERP
  • 2010s: 软件放大 "注意力" — social, mobile
  • 2025+: AI 放大 "认知带宽" — LLM, agent, copilot

⚠️ 数量 vs 杠杆

以前 10x 增长 = 招 10x 人 (HR, 管理, 文化成本)。 现在 10x 增长 = 个人 × agent (杠杆), headcount 反而可能 ↓。 "people-first" 文化被 "leverage-first" 文化取代。

✅ 新铁三角

人类 × 工具 × 流程 的旧铁三角, 现在 人类 × AI 协作层 × 治理层。 新铁三角的 3 个核心问题:

  • 协作层: 哪个任务给 agent, 哪个留人?
  • 治理层: 谁 audit agent 输出?
  • 决策层: 谁决定架构, 节奏频率多少?

三条主线趋势 (8 源 cross-validated)

A. 极小化 + 高密度

[TO VERIFY] Cognition (Devin) 几十号员工 = 几亿美元 ARR; OpenAI ~4000 人; Anthropic ~1000 人却估值 $300B+

单兵能力 = 一辈子工程师 / 领域专家 / AI 编排者 一种。

内部组织无正式层级: "no managers" 不完全准 — 没有 middle management 但有 IC ladder + 出 owner 文化。

B. 中层被 AI agent 接管

不是 "中层消失", 是 "中层职责被 AI agent 接管": 状态同步 / 跨团队对齐 / 写周报 / 做安排 / 走流程 / 跑 vendor。

留下的是 判断 + 风险 + 跨人化关系 编进 code review / quarterly review / governance board。

关键: 中层重构 = trainer (给 agent 写 prompt/eval/RAG) + audit owner (看 agent 输出哪里错), 不是裁员。

[vault verified] Greg Isenberg 06-26: "Pick one [new skill] and get dangerous. Pick two and you have leverage. Pick three and you become the kind of person that everyone wants on the team..."

C. 市场 / 销售 / 设计 / 初级代码 = AI-first

高度流程化工种: 第一份工作量都靠 AI 完成, 人类 = override + 偶尔手工。

探索型工种 (调研 / 外联 / 竞品 / 客户开发): 半数时间跟 agent 协作, 半数时间推关键判断。

[vault verified] Tomasev (DeepMind): N≥1000 agent 时 "协调 / 信任 / 价值观对齐" 比 "能力本身" 更紧要。

AGI 后的 4 种结构原型

A. AI-Native Pyramid

    顶层:  CEO + 3-5 总监 (纯判断 + 高风险决策 + 投资人/大客户关系)
   中层:  每个部门 1-2 个 AI-Director (≈ 以前 5 个中层, 现在 1 个看 agent 状态)
   执行层:  开发者/设计师/AI Trainer + 各自 AI 助理
   Agent Squad: 销售 / 客服 / 法务 / 财务 / BD / 调研 各 3-10 个 agent
                 (命名/评估/被监控) — Greg Isenberg 06-26 命名

关键机制: 每个执行者跑 5-10x 之前的项目包, job ladder 从 "管 5 人" 变成 "编排 5 个 agent"。

现实案例: [TO VERIFY] Anthropic / OpenAI / Cognition 内部部门结构。

B. Co-founder Pod + Agent Cloud

   核心:   1-3 founder (不分工, 谁擅长谁干 + 决策非同步)
   Pod:    2-5 个 AI-resilient 工程师 + 各 自带 custom config 的 agent
           (几乎人人 "独立产线" — Greg "Builder Distributor")
   Outside:  RPA bot 自动 run bookkeeper / vendor / legal NDA / bot 客服

关键机制: 没有 "经理" 这件事 — 决策靠每周 founder call + 异步 weekly update + 自动 dashboard。

现实案例: [TO VERIFY] Lovable / Linear / 独立工程师 + AI 框架 公司。

C. AGI Factory (2027-2030 真正成型)

   Orchestrator (1) = 真人, 负责 "什么值得做"
   Design Loop   (2) = 真人, "设计 agent 该如何写 prompt / 装什么 tool"
   Eval Loop     (3) = 真人 + agent, "给 agent 评分 / 改错 / 下个版本"
   Architecture  (4) = 真人 + agent, "每天重新设计公司里的岗位"
   Meta-Audit    (5) = 真人, "这个公司本身的组织架构对不对"

[vault verified] TheAIGRID 07-14 + Kokotajlo 07-13: 人类从 "执行者" 变 "meta-architect"。

[vault verified] Kokotajlo: "在 2026-2027 维持 distinction 'what to build vs how to build' 比以前更值钱; 老板的真正技能 = 提出正确问题 + 知道哪些 agent 输出能信"。

D. Trainer + Auditor + Owners (现实迁移路线)

   一线员工   (60%) = 从事 "AI 做不了的事" (判断/关系/创意)
   AI Trainer (15%) = 专门负责 "给公司 agent 写 prompt、建构 eval、出错修补"
   AI Auditor (10%) = 监督 agent 输出、监督 AI Trainer、决定 release/rollback
   Owners     (15%) = 真人部门负责人, 只对结果负责

关键洞察: AI Trainer 跟 AI Auditor 是 新生成的两类角色, 不是 "中层换皮"。这两类必须是单独的 job family, 用 IC ladder 升。

现实案例: [TO VERIFY] BCGI 2024 / McKinsey March 2025 "重塑职能" 报告里的样本案例。

  • 每个 agent 都有名字 + 上级 owner + eval 标准, 跟传统 employee 一样治理

关键风险 / 反方

"AGI 改变组织架构" 真反方极少, 但 boundary conditions 重要:

1. "No manager" 实际不行

跨人关系处理 / 信任 + 责任归属 / 危机时刻方向判断 — 这些纯靠 "异步 + agent" 不够。

任何真大型 org 都需要 "人形的 owner 网络"。

2. AI bias / 合规

[vault verified] Tomasev: agent N≥1000 后, 价值对齐问题爆炸。

原型 D 里的 AI Auditor 角色不是 nice-to-have, 是 P0。

3. Agent 自身维护成本被低估

跟 "雇人" 不同, agent 维护需要 perpetual eval + retrain + prompt up-version — "complexity tax"。

人越少, agent 维护的人均成本越高。1 人 + 10 agent 可能比 5 人 + 2 agent 贵。

4. McKinsey 2025 数据缺口

[TO VERIFY] 大部分 "AI 重塑组织" 还在宣传期, 实证 ROI 数据尚少; 5/10 公司报告 "agent 项目失败" 的根因不是能力, 是 "owners 没换好岗"。

5. Tristan Harris 视角

[vault verified] 人类从 "执行者" → "meta-architect", 这层转换对 99% 人力是 demotion 不是 promotion。

社会级别的后果: 大量中产白领失业 + 政府介入 + 反 AI 海啸。

公司层面活得很好, 社会层面很惨 — Patrick 经商时这是 "supportive politician + 安全网建设" 信号。
