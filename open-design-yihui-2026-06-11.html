Claude Design vs Open Design

❌ Claude Design 问题

  • Token 限额极小
  • $200/月套餐也开发不了几个项目
  • 无法接入自己惯用的 Agent
  • 设计系统和 Skills 封闭

✅ Open Design 优势

  • Token 完全自控（用自己订阅的 Agent）
  • 支持 Cloud Code / CodeX / Cursor Agent / i-Mesh
  • 更丰富开放的设计系统和 Skill
  • 免费开源

安装与配置流程

1下载 Open Design
前往 getopendesign.com → Download → MacOS / Windows 版本，解压安装
2选择本地 COI（Agent）
首次启动时选择本地 Agent 类型：Cloud Code / CodeXCO / Open Code / Cursor Agent / i-Mesh。安装对应插件后即可使用套餐额度，而非消耗 Open Design 自己的 Token
3了解设计系统概念
设计系统 = 乐高说明书 + 标准零件包，不管谁来搭，最终风格都统一。Open Design 提供的是「快速起步的风格模板」，不是完整企业设计系统

设计系统核心概念

「如果说我们做产品像搭乐高的话，那设计系统就像那些乐高的一个说明书和标准的一个零件包。有了它，不管谁来搭，最终搭出来的风格都是统一的。」

Open Design 工作流

1选择页面 Section
Section = 页面组成部分（Hero、Footer、FAQ、Pricing、Steps 等）。最多选 5 个。Open Design 会据此生成对应的设计系统
2AI 生成设计系统
生成品牌颜色、组件样式（Button、Card、表单、Layout 等）。还原度很高，包含登录页面和后台管理页面，并支持响应式
3复制代码到项目
将生成的设计文件复制到实际项目目录中，即可在 Cursor Agent 中查看和进一步修改
4用 Cursor 查看和调试
推荐用 Cursor Agent 打开项目，可以直接在 IDE 中调试生成的代码

关键技巧：组件封装的重要性

💡 最重要的一个认知

如果没有组件封装，AI 会在多个页面重复写同一个组件。比方说三个页面都有价格看板，当价格从 $15 改成 $20 时，AI 大概率会漏改其中一些页面。但如果使用了共享组件，只改一处，所有页面自动同步更新。

封装规则示例（Agent Sam）：

  • 共享组件 — 价格看板等重复元素应封装为独立组件
  • 一处修改，处处生效 — 改 $15→$20，所有页面自动同步
  • AI 自动封装 — 页面膨胀时，Agent 会自动封装新出现的重复元素
  • Temp 文件夹 — 不想提交到 GitHub 的实验代码，放进 temp 文件夹，GitHub 不会追踪

使用体验总结

  • Cloud Code + OpenCity 效果最好 — 设计出来的样式更好看
  • CodeX Token 给得很足 — 社区很多用户已经用 CodeX
  • Cursor Agent 适合调试 — 查看、修改、预览生成的代码
  • Design System 页面由 AI 生成 — 目前只包含基础组件（Button、Card、表单、Layout），更多组件需手动让 AI 继续加入
